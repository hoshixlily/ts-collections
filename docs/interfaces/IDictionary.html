except

  • except(enumerable, comparator?, orderComparator?): IEnumerable<KeyValuePair<TKey, TValue>>

  • Produces the set difference of two sequences by using the specified equality comparer or order comparer to compare values.

    About the difference between comparator and orderComparator:

    • If both comparator and orderComparator are specified, the order comparator will be used for internal operations.
    • If only one of the comparators is specified, the specified comparator will be used for internal operations.
    • If no comparator is specified, it will use the default equality comparer.

    If the elements of the enumerable can be sorted, it is advised to use the orderComparator due to its better performance.

    Example:

        var numberList1 = new List([1, 2, 2, 3, 3, 3, 4, 5]);
        var numberList2 = new List([2, 5, 5, 6, 7, 8, 8]);
        var result = numberList1.except(numberList2).toArray(); // [1, 3, 4]

    Parameters

    • enumerable: IEnumerable<KeyValuePair<TKey, TValue>>

      The enumerable sequence whose distinct elements that also appear in the first sequence will be removed.

    • Optional comparator: null | EqualityComparator<KeyValuePair<TKey, TValue>, KeyValuePair<TKey, TValue>>

      The comparator function that will be used for equality comparison. If not provided, default equality comparison is used.

    • Optional orderComparator: null | OrderComparator<KeyValuePair<TKey, TValue>, KeyValuePair<TKey, TValue>>

      The comparator function that will be used for order comparison. If not provided, default equality comparison will be used.

    Returns IEnumerable<KeyValuePair<TKey, TValue>>

    Throws

    If the enumerable is null or undefined.
